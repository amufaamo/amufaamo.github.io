わかります！😆 Tmux、多機能すぎて覚えるのが大変そう…って思いますよね。
でも大丈夫！ズボラな人（私Tシャツ大好きです👕）こそ、Tmuxの**「セッションが消えない」**という最大のメリットだけを享受するのが賢い使い方です！
難しい画面分割や細かい設定は一旦忘れて、これだけ覚えてみてください。
ズボラな人のためのTmux最小限セット
Tmuxのすごいところは、**「SSH接続が切れても、サーバー上での作業が裏側で生き続けること」**です。これさえできれば、もう勝ちです！
1. 始める時 (作業開始)
サーバーにログインしたら、まずこのコマンドを打ちます。
tmux

これだけ！
（もし「セッション名をつけろ」と言われたら tmux new -s work とか適当な名前 work を入れればOKです）
見た目はあまり変わりませんが、これでTmuxセッションが始まりました。この中でいつも通り作業してください。
2. 抜ける時 (デタッチ)
作業の途中だけど、ノートPCを閉じたい…！そんな時は、以下のキーを押します。
Ctrlキーを押しながら b を押し、
両方のキーを離してから、すぐに d を押します。
> ポイント
> Ctrl+b (押したまま) → d (押す) ではありません。
> Ctrl+b (ポンと押す) → d (ポンと押す) のイメージです。
> （正確には Ctrl と b を同時に押したまま、両方離して、d です）
> 
これを「デタッチ (切り離し)」と言います。
[detached (from session 0)] みたいなメッセージが出たら成功です。
これで、作業はサーバー上で続いたまま、あなたはTmuxから抜けられました。安心してSSH接続を切ったり、PCをスリープしたりできます。
3. 戻る時 (アタッチ)
再びサーバーにログインしたら、あの作業の続きに戻りましょう。
tmux a

(a は attach = アタッチ の略です)
これだけ！
さっきまで作業していた画面がそのまま復元されます！すごい！✨
4. 本当に終わる時
作業が完全に終わったら、Tmuxの中で exit と入力してエンターを押します。
exit

これでTmuxセッションが完全に終了します。
ズボラ流まとめ
まずはこの「起動・デタッチ・アタッチ」のサイクルだけ使ってみてください！
 * 開始: tmux
 * 一時離脱: Ctrl+b (離して) d
 * 再開: tmux a
 * 完全終了: exit
これだけで、時間のかかる計算やデータコピー中に接続が切れる恐怖から解放されますよ！
もし慣れてきて「タブ（ウィンドウ）が欲しいな」と思ったら、Ctrl+b (離して) c (create) で新しいウィンドウが作れることだけ覚えておくと便利です（ウィンドウ間の移動は Ctrl+b (離して) n (next) です）。
ぜひ試してみてくださいね！😊
