開発に集中しているとき、わざわざGitHubのサイトを開いて「New Repository」ボタンを押すのって、ちょっと面倒じゃありませんか？

「今のこのコードを、今すぐGitHubに上げたい！」

そんな願いを叶える、**黒い画面（ターミナル）から一歩も出ずにリポジトリ作成からPushまで完了させる方法**をまとめました。これを使えば、開発フローがもっとスマートになりますよ！

### 事前準備：GitHub CLI (`gh`) の導入

この記事では、GitHub公式のコマンドラインツール **GitHub CLI (`gh`)** を使用します。
まだ入れていない方は、MacならHomebrewでサクッと入れておきましょう。

```bash
brew install gh

```

インストール後、以下のコマンドでログイン認証を済ませておいてくださいね。

```bash
gh auth login

```

---

### 手順はたったの3ステップ！

すでに手元にコードがあるディレクトリ（フォルダ）にいる前提で進めます。

#### 1. Gitの初期化とコミット

まずは基本のGit初期化です。まだGit管理していない場合はここからスタート。

```bash
git init
git add .
git commit -m "Initial commit"

```

#### 2. リポジトリ作成＆リモート登録＆Push（魔法のコマンド）

ここが今回のハイライトです！
通常はブラウザでリポジトリを作って、URLをコピーして、`git remote add...` とやりますよね。

`gh` コマンドなら、**この1行だけで全て終わります。**

```bash
# 公開リポジトリ (Public) の場合
gh repo create --public --source=. --remote=origin --push

# 非公開リポジトリ (Private) の場合
gh repo create --private --source=. --remote=origin --push

```

#### コマンドの解説

呪文のように見えますが、オプションの意味はとてもシンプルです。

* `--public` / `--private`: リポジトリの公開設定。
* `--source=.`: **「現在のディレクトリ (.)」** をリポジトリのソースとして使いますよ、という指定。これがないと空のフォルダが作られてしまいます。
* `--remote=origin`: ローカルのGitに、リモートリポジトリを `origin` という名前で登録します。
* `--push`: 作成と同時に、今のコミットをプッシュします。

---

### もし「リポジトリ名」を指定したい場合は？

上記のコマンドだと、**現在のディレクトリ名**がそのままGitHubのリポジトリ名になります。
もし、ディレクトリ名とは違う名前を付けたい場合は、引数を渡してあげましょう。

```bash
# "my-new-app" という名前でPrivateリポジトリを作成
gh repo create my-new-app --private --source=. --remote=origin --push

```
