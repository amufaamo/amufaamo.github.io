こんにちは！
今日は、テレワークや出張先での作業が劇的に快適になる**Tailscale（テールスケール）**の活用術をシェアします。

「自宅から、職場のLAN内にあるサーバー（Linuxなど）を触りたい」
「でも、VPNの設定は面倒だし、セキュリティも心配…」
「職場にいる時と自宅にいる時で、接続先設定を変えるのが手間…」

そんな悩み、ありませんか？
**Tailscaleの「Subnet Router（サブネットルーター）」機能**を使えば、職場のPC（WindowsでもMacでもOK！）を中継地点にして、自宅から職場のサーバーへ、**まるで職場のデスクにいるかのように**アクセスできるようになります。

一度設定してしまえば、**「職場でも自宅でも、全く同じ設定で繋がる」** という魔法のような環境が手に入りますよ！

## 今回の構成とゴール

やりたいことはこれです。

* **ターゲット:** 職場のLinuxサーバーなど（ここにVS Codeで繋ぎたい）
* **中継役:** 職場の常時起動PC **（Windows / Mac どちらでもOK）**
* **操作端末:** 自宅や出先のPC **（Windows / Mac どちらでもOK）**

Tailscaleを全端末に入れるのではなく、**職場のPC**に「ここから先のサーバーへの通信は、私が案内しますよ！」という**ルーター（案内係）**の役割を持たせるのがポイントです。

## 手順1：職場のPC（中継役）の設定

まず、職場にある「中継役」となるPCの設定です。Tailscaleは既にインストール済みとします。
ここでやるのは、「特定のIPアドレス（サーバー）への通信を中継するぞ」という宣言です。OSによってコマンドが少し違います。

### A. 職場のPCが「Windows」の場合

1. **PowerShell**を検索し、**「管理者として実行」**します。
2. 以下のコマンドを入力します。

```powershell
tailscale up --advertise-routes=[サーバーのIPアドレス]/32 --reset

```

### B. 職場のPCが「Mac」の場合

1. **ターミナル**を開きます。
2. 以下のコマンドを入力します（パスワードを求められたらMacのログインパスワードを入力）。

```bash
sudo tailscale up --advertise-routes=[サーバーのIPアドレス]/32 --reset

```

> **解説:**
> * `[サーバーのIPアドレス]` は、繋ぎたいサーバーの実際のIP（例: `192.168.1.10` や `133.x.x.x` など）に書き換えてください。
> * `/32` をつけると、「その特定のサーバー1台だけ」を中継対象にします。安全でおすすめです。
> 
> 

## 手順2：Tailscale管理画面での承認

PC側で宣言しただけではまだ動きません。Webの管理画面で「許可」を出しましょう。これはOS共通の作業です。

1. [[TailscaleのAdmin Console](https://login.tailscale.com/admin/machines)](https://login.tailscale.com/admin/machines)にアクセスします。
2. 手順1で設定した「中継役のPC」を探し、右側の `...` メニューをクリック。
3. **"Edit route settings"** を選択します。
4. 設定したIPアドレスが表示されているので、**チェックを入れて有効化（オン）**にし、Saveします。

これで、Tailscaleネットワーク全体が「あ、このサーバーに行くならあのPCを通ればいいんだな」と理解しました！

## 手順3：自宅PC（操作端末）のSSH設定

最後に、手元のPC（自宅のMacやWindows）の設定です。
ここが一番の感動ポイント。Subnet Routerが効いているので、設定ファイル（`~/.ssh/config`）には**職場のサーバーの本来のIPアドレス**をそのまま書くだけでOKなんです。

### 1. 秘密鍵の準備

職場で使っているSSHの秘密鍵（`id_rsa`など）を、USBメモリなどで安全に自宅PCへコピーしてきます。
今回は例として `~/.ssh/id_rsa_work` という名前にリネームして保存したとします。

### 2. 権限の修正（Mac/Linuxの場合）

Macをお使いの場合、鍵ファイルを持ってきただけだと「権限がゆるすぎる」と怒られて接続できないことがあります。ターミナルで以下を実行しておきましょう。

```bash
chmod 600 ~/.ssh/id_rsa_work

```

*(Windowsの場合は、ユーザーフォルダの `.ssh` フォルダに置けば基本的に大丈夫です)*

### 3. SSH Configの設定

VS Codeなどで `~/.ssh/config` を開き、以下のように追記します。

```ssh
Host my_work_server
  HostName [サーバーのIPアドレス]
  User [サーバーのユーザー名]
  IdentityFile ~/.ssh/id_rsa_work

```

見てください、`HostName` に Tailscale用のIP（100.x.x.x）ではなく、**職場のサーバーのIP（ローカルIPやグローバルIP）をそのまま書いています。**

## 完成！

これで、自宅のターミナルやVS Codeから：

```bash
ssh my_work_server

```

と打つだけで、自動的に職場の中継PCを経由してサーバーに繋がります！

### この設定のすごいところ

ノートPCを職場に持っていった時はどうなるの？と思いますよね。
実は、**そのままで大丈夫**なんです。

* **自宅（Tailscale ON）：** 中継PCを経由してサーバーへ
* **職場（Tailscale OFF/ON）：** 職場のLANから直接サーバーへ

自動的に最適なルートが選ばれる（または直接通信できる）ので、いちいち「自宅用設定」「職場用設定」と書き換える必要がありません。
