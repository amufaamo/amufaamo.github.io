普段、解析サーバーへデータを転送するときやログインするとき、こんな風にコマンドを打っていませんか？

```bash
scp ./data.txt user_name@192.168.11.25:~/

```

毎回この長いIPアドレス（しかもユーザー名まで！）を打つのは、指も疲れますし、「あれ、アドレス何番だっけ？」となったり、タイプミスでエラーが出たりして地味にストレスですよね。

今回は、そんな悩みを一瞬で解決する **`~/.ssh/config`** の設定方法をご紹介します。これ設定すると、世界が変わりますよ！

#### 1. 何ができるようになるの？

この設定をすると、IPアドレスの代わりに **「自分で決めた好きな名前」** でアクセスできるようになります。

例えば、`myserver` という名前を付ければ、

**Before:**

```bash
scp ./data.txt user_name@192.168.11.25:~/

```

**After:**

```bash
scp ./data.txt myserver:~/

```

これだけでOKになります！もちろん `ssh myserver` でのログインも可能です。劇的に短くなりますよね。

#### 2. 設定手順

設定はとても簡単です。ホームディレクトリにある `.ssh` フォルダの中に `config` というファイルを作るだけです。

**手順①：ファイルを開く（または作成）**

ターミナルで以下のコマンドを打ちます。

```bash
nano ~/.ssh/config

```

**手順②：設定を書き込む**

ファイルの中に、以下の形式で情報を記述します。

```text
Host myserver
    HostName 192.168.11.25
    User user_name
    Port 22
    IdentityFile ~/.ssh/id_rsa

```

* **Host**: ここに「呼びたい名前」を書きます（例：`lab-pc`, `aws`, `myserver` など）。これが短縮コマンドになります。
* **HostName**: ここに実際の「IPアドレス」を書きます。
* **User**: ログインする「ユーザー名」です。
* **IdentityFile**: 秘密鍵認証を使っている場合は、鍵のパスを書きます（パスワード認証ならこの行は不要です）。

**手順③：権限を変更する（重要！）**

保存したら、最後にファイルの権限を変更します。これをしておかないと、SSHコマンドが「セキュリティが甘い！」と怒って実行してくれません。

```bash
chmod 600 ~/.ssh/config

```

#### 3. さいごに

たったこれだけの設定で、毎日の作業が少しだけ、でも確実に快適になります。
特に複数のサーバーを使い分けている方には必須の設定です。

IPアドレスが変わった時も、このファイルを書き換えるだけで済むので管理も楽になりますよ。ぜひ試してみてください！
