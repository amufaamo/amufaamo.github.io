ターミナル作業を効率化するために **tmux** を使い始めたものの、こんなことになっていませんか？

「ログを確認したいのに、マウスホイールを回しても履歴が戻らない…！」
「矢印キーを押してもコマンド履歴が出るだけで、画面がスクロールしない💦」

これ、地味にストレスですよね。私も最初は「えっ、壊れた！？」と焦りました（笑）。

でも大丈夫です！実は設定ファイルを **たった1行** 書き換えるだけで、いつものターミナルのようにマウスでスルスルとスクロールできるようになりますよ。

今回は、tmux生活が一気に快適になる魔法の設定をご紹介します✨

---

### 解決策：設定ファイルに「マウス有効」を記述する

結論から言うと、ホームディレクトリにある `.tmux.conf` という設定ファイルに、以下の1行を追加するだけです。.tmux.confがなければ作ってください。

#### 1. 設定ファイルを開く（または作成する）

まだファイルがない場合も、このコマンドでOKです。`vim` や `nano` などお好みのエディタで開いてください。

```bash
nano ~/.tmux.conf

```

#### 2. 以下のコードを追加する

```bash
set -g mouse on

```

これだけです！
以前のバージョン（2.0系など）では設定が複雑でしたが、今の新しいバージョンならこの1行だけで、以下のことができるようになります。

* **マウスホイールでのスクロール**
* **マウスでのウィンドウ（ペイン）選択**
* **マウスドラッグでのペインサイズ変更**

まさに「これを待ってた！」という機能ですよね。

---

### 重要：設定を反映させる

ファイルを保存しただけでは、まだ機能は有効になりません。
tmux上で以下のコマンドを実行して、設定を再読み込みさせましょう。

```bash
tmux source-file ~/.tmux.conf

```

エラーが出ずにコマンドが通れば成功です！
さっそくマウスホイールを回してみてください。「動いた！」という感動が待っているはずです✨

---

### おまけ：キーボードだけでスクロールしたい時は？

「俺はマウスなんて使わない、キーボード派だ！」という硬派な方は、**コピーモード** を使いましょう。

1. **`Ctrl` + `b**` を押して指を離す（プレフィックスキー）
2. **`[`** を押す

これで画面右上に `[0/0]` のような表示が出て、矢印キーや `PageUp` / `PageDown` でスクロールできるようになります。
元のモードに戻る時は **`q`** を押せばOKです。

---

### まとめ

* `~/.tmux.conf` に `set -g mouse on` を書く
* `tmux source-file ~/.tmux.conf` で反映する

たったこれだけで、tmuxの操作性が劇的に向上します。
スクロールのストレスから解放されて、快適なターミナルライフを送ってくださいね！
