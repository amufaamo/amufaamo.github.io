# 1. はじめに

* **悩み:** 複数のサーバー（解析用、開発用、本番用など）をVS CodeのRemote SSHで繋いでいると、「今どっちの環境だっけ？」とヒヤッとしたことはありませんか？
* **リスク:** 間違えて本番環境でコマンドを打ってしまったら大変！
* **解決策:** 拡張機能「Peacock」を使って、サーバーごとにウィンドウ枠の色を変えちゃいましょう。

# 2. Peacockとは？

* John Papa氏が開発した、VS Codeのワークスペースごとに色設定を簡単に変えられる拡張機能。
* `settings.json` を手動でいじらなくても、直感的に色が選べるのが魅力。

# 3. 使い方

1. 拡張機能マーケットプレイスから「Peacock」をインストール。
2. **【重要】ここがポイント！**
* まずは**対象のリモートサーバーにSSH接続**します（または対象のワークスペースを開きます）。
* *※ここを強調！「ただVS Codeを開いただけだとコマンドが出てこないことがあるので注意」と書くと、読者の「できない！」を防げます。*


3. コマンドパレット（`F1`）で `Peacock: Change to Favorite Color` を選択。
4. 好きな色を選ぶだけで、パッと色が変わります！

# 4. 実際の使用例（Before/After）

* **Local:** デフォルトの黒
* **Dev Server:** 緑（安心）
* **Prod Server:** 赤（警戒！）
* 画像があると「こんなに分かりやすくなるんだ！」と一目で伝わります。（さっきのスクリーンショットも使えそうですね！）


# 5. あれ？Peacockが動かない！時の対処法

「手順通りにやったのに色が選べない…」「コマンドが出てこない…」
そんな時は、焦らず以下のポイントをチェックしてみてください。

#### 1. 「ファイル」ではなく「フォルダ」を開いていますか？

これが一番多い原因です！Peacockは設定をそのディレクトリの `.vscode/settings.json` というファイルに保存します。
そのため、単一のファイルだけを開いている状態だと設定が保存できません。

* **NG:** SSH接続して、特定のスクリプトファイルだけを開いている。
* **OK:** 「フォルダを開く（Open Folder）」で、作業ディレクトリ全体を開いている。

#### 2. リモート側にインストールされていますか？

Remote SSHを使っている場合、拡張機能が「自分のPC（ローカル）」には入っていても、「接続先のサーバー（リモート）」に入っていないことがあります。

* 拡張機能のタブを開いて、Peacockの欄に**「SSH: [サーバー名] にインストール」**というボタンが出ていませんか？
* もし出ていたら、ポチッと押してインストールしてあげてください。

#### 3. 【裏技】拡張機能なしで設定する方法（手動設定）

「会社のポリシーで勝手に拡張機能を入れられない…」「どうしてもPeacockが動かない！」
そんな場合も諦めないで！VS Codeの標準機能だけで同じことができます。（実はPeacockも裏ではこれを自動でやっているだけなんです）

1. コマンドパレット（`F1` または `Ctrl+Shift+P`）を開く。
2. `Preferences: Open Workspace Settings (JSON)` と入力して選択。（日本語なら「基本設定: ワークスペース設定を開く(JSON)」）
3. 開いたファイル（`.vscode/settings.json`）に、以下の魔法のコードをコピペしてください。

**▼ コピペ用コード（例：本番環境用の赤色アラート！）**

```json
{
  "workbench.colorCustomizations": {
    "titleBar.activeBackground": "#ff0000",      // 上のバーの背景色（真っ赤！）
    "titleBar.activeForeground": "#ffffff",      // 上のバーの文字色（白）
    "titleBar.inactiveBackground": "#ff000099",  // 非アクティブ時の背景色
    "statusBar.background": "#ff0000",           // 下のバーの背景色
    "statusBar.foreground": "#ffffff"            // 下のバーの文字色
  }
}

```

これで拡張機能なしでも、誤操作防止の色分けが完璧にできちゃいます！
`#ff0000` の部分を好きなカラーコードに変えれば、自分好みの色にカスタマイズも可能ですよ。


# 5. まとめ

* 視覚的に区別することで、無意識の誤操作を劇的に減らせます。
* 特に複数の解析サーバーを行き来するバイオインフォマティクスなどの研究用途にもおすすめ！
