# はじめに

Firebaseのプロジェクトを削除するとき、わざわざコンソール画面（ブラウザ）を開いて、設定の奥深くへ進んで……という作業をしていませんか？

実は、**Firebase CLI**を使えば、ターミナルからコマンド一発で削除ができちゃうんです。
GUIもわかりやすいですが、慣れるとCLIの方が圧倒的に早いですよ！

今回はその手順をメモしておきます。

---

# 手順1：プロジェクトIDを確認する

まずは、削除したいプロジェクトの「Project ID」を確認します。
間違ったプロジェクトを消さないように、必ず一覧を出して確認しましょう。

```bash
firebase projects:list

```

これを実行すると、現在アクセス可能なプロジェクトの一覧が表形式で表示されます。
**Project ID** の列にある文字列（例: `my-test-app-12345`）をコピーしてください。

# 手順2：削除コマンドを実行する

IDがわかったら、以下のコマンドで削除を実行します。

```bash
firebase projects:delete <ここにプロジェクトID>

```

例えば、IDが `my-test-app-12345` の場合はこうなります。

```bash
firebase projects:delete my-test-app-12345

```

# 手順3：最終確認

コマンドを打つと、以下のような警告が表示されます。

> **warning:** YOU ARE ABOUT TO DELETE A GOOGLE CLOUD PROJECT. THIS IS IRREVERSIBLE.

「本当に消していいの？ 元に戻せないよ？」と聞かれるので、問題なければ **Y** を入力して **Enter** を押してください。

これで削除完了です！🎉

---

# 注意点

* **権限:** この操作を行うには、そのプロジェクトの**オーナー（Owner）権限**が必要です。
* **慎重に:** CLIは手軽な分、うっかりミスも起きやすいです。コマンドを実行する前に、IDが合っているか指差し確認することをおすすめします。
