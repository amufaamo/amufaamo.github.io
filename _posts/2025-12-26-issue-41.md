# 1. はじめに

* **悩み:** 複数のサーバー（解析用、開発用、本番用など）をVS CodeのRemote SSHで繋いでいると、「今どっちの環境だっけ？」とヒヤッとしたことはありませんか？
* **リスク:** 間違えて本番環境でコマンドを打ってしまったら大変！
* **解決策:** 拡張機能「Peacock」を使って、サーバーごとにウィンドウ枠の色を変えちゃいましょう。

# 2. Peacockとは？

* John Papa氏が開発した、VS Codeのワークスペースごとに色設定を簡単に変えられる拡張機能。
* `settings.json` を手動でいじらなくても、直感的に色が選べるのが魅力。

# 3. 使い方

1. 拡張機能マーケットプレイスから「Peacock」をインストール。
2. **【重要】ここがポイント！**
* まずは**対象のリモートサーバーにSSH接続**します（または対象のワークスペースを開きます）。
* *※ここを強調！「ただVS Codeを開いただけだとコマンドが出てこないことがあるので注意」と書くと、読者の「できない！」を防げます。*


3. コマンドパレット（`F1`）で `Peacock: Change to Favorite Color` を選択。
4. 好きな色を選ぶだけで、パッと色が変わります！

# 4. 実際の使用例（Before/After）

* **Local:** デフォルトの黒
* **Dev Server:** 緑（安心）
* **Prod Server:** 赤（警戒！）
* 画像があると「こんなに分かりやすくなるんだ！」と一目で伝わります。（さっきのスクリーンショットも使えそうですね！）

# 5. まとめ

* 視覚的に区別することで、無意識の誤操作を劇的に減らせます。
* 特に複数の解析サーバーを行き来するバイオインフォマティクスなどの研究用途にもおすすめ！
